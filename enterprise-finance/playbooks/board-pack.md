# Playbook: Board Pack — Deck, Investor Update, BvA Package

Three artifacts, one rule above all: **the same metric never appears under two definitions,
and definitions never change silently between periods.** Experienced board members always
catch a quietly redefined number — and it costs more credibility than a bad quarter.
If a definition must change, disclose it on the slide where it appears and restate prior
periods under the new definition.

Cadence rules: quarterly board deck sent **48–72 hours before the meeting** — the meeting is
discussion, not readout. Monthly investor update within 10 business days of close. BvA package
within 10 business days of close.

## 1. Quarterly board deck (8 sections + appendix)

1. **CEO summary** — highlights, lowlights, asks. One slide. Lowlights are mandatory; a deck
   without them reads as either dishonest or unaware.
2. **KPI dashboard vs. plan** — ARR, NRR/GRR, burn multiple, cash + runway (3-mo avg basis),
   headcount. Actual | plan | delta. Definitions footnoted (pointer to the definitions block).
3. **Financial review** — summary P&L vs. budget; **ARR bridge** (see format below); BvA
   commentary for material variances (timing vs. permanent).
4. **GTM deep-dive** — pipeline coverage (3–4x standard), win rates, CAC payback by segment,
   rep productivity vs. ramp plan.
5. **Product / roadmap** — shipped vs. committed, next quarter's bets, one slide of usage
   evidence.
6. **People** — headcount actual vs. plan, key hires/departures, regretted attrition if any.
7. **Strategic topics for discussion** — **the actual point of the meeting.** 2–3 topics,
   each framed as: context (half a slide) → options with trade-offs → the specific decision
   or input being asked of the board. Never status disguised as strategy.
8. **Formal items** — approvals needed: option grants, budget, 409A refresh, officer matters.
   (Board decides; the deck only preps.)

**Appendix** — full financial statements, cohort/NRR triangle, detailed pipeline, metric
definitions block.

### ARR bridge format (section 3 and the monthly waterfall)

| Month | Beginning ARR | + New | + Expansion | − Contraction | − Churned | = Ending ARR |
|-------|--------------|-------|-------------|---------------|-----------|--------------|

By segment where segments exist. Cohort NRR triangle in the appendix (rows = start cohort,
columns = months since start, cells = % of starting ARR retained+expanded).

## 2. Monthly investor update (email — one screen, consistent month over month)

```markdown
Subject: <Company> — <Month YYYY> update

**TL;DR:** <3 bullets — the quarter's truth in 30 seconds>

| KPI | This month | Last month | Plan |
|-----|-----------|-----------|------|
| ARR (growth % MoM) | | | |
| Net burn (3-mo avg) | | | |
| Cash / runway (months) | | | |
| NRR (T12M) | | | |
| Headcount | | | |

**Highlights** — <3-5 bullets, concrete: logos, ships, numbers>
**Lowlights** — <2-4 bullets. Candor is the credibility signal. "None" is a red flag — if the
draft says none, push back until something real is on the list.>
**Product / GTM notes** — <2-3 bullets>
**Asks** — <specific, actionable: intros to X, candidates for Y, advice on Z. The asks are
why investors read updates.>
**Thank-yous** — <who helped since last update>
```

Same KPI rows, same definitions, every month. If a number was restated, say so in-line.

## 3. Budget vs. Actuals (BvA) package

Prerequisite: a **frozen budget** (board-approved annual plan). The budget never moves; the
rolling forecast is the living view. **Re-baselining the budget monthly so BvA always looks
fine is forecast theater — refuse to do it and say why.**

Contents:

1. **Summary P&L**: line | actual | budget | $ variance | % variance | YTD actual | YTD budget.
2. **Variance commentary table** — for anything beyond ±10% or ±$10–25K (calibrate the dollar
   threshold to stage): line | driver of variance | **timing vs. permanent** | action.
   Permanent variances flow into the reforecast; timing variances get an expected catch-up
   month. "What driver actually explains the miss — pipeline, win rate, or ACV?" is the
   standard Henrik must meet: name the driver, not the symptom.
3. **Headcount actual vs. plan** — starts, slips, and the burn impact of each slip.
4. **Updated full-year forecast vs. budget** — the honest landing estimate.

## Close-quality gate (Luca — before any board number ships)

The deck's data comes from a locked close, or it carries a banner. Checklist (business days
5–10): bank/credit-card recs → revenue recognition + deferred revenue waterfall tied to the
billing system → AR review and dunning status → accruals (payroll, bonus, commissions) →
prepaid amortization → stock comp booked → **flux analysis: every >10% MoM P&L movement
explained** → lock. No "temporary" journal entries older than one period.

If the close isn't done: banner every affected page `DATA AS OF <date> — PRE-CLOSE, SUBJECT
TO ADJUSTMENT`, and list which lines are most likely to move.

## Quality bar for the whole pack

- Every number traceable: board number → model/close source. No orphan numbers.
- Benchmarks cited with source + year, flagged "verify current via WebSearch — these date fast."
- Runway shown on 3-month-average net burn, base and downside.
- Asks are specific enough that an investor can act on them from their phone.
